# ProjectFlow — Система управления проектами

Этот проект представляет собой веб-приложение для организации работы над задачами по методологии Kanban. Идея заключалась в создании упрощённого аналога популярных инструментов вроде Trello или Jira, который можно использовать для управления личными проектами или небольшими командными задачами в рамках учебной деятельности. Тема была выбрана потому, что подобные системы востребованы в реальной разработке, и их создание позволяет познакомиться с полным циклом разработки веб-приложения — от проектирования базы данных до реализации интерактивного пользовательского интерфейса.

## Технологии и архитектура

Приложение построено по классической клиент-серверной архитектуре. Для серверной логики использовался Node.js с фреймворком Express, который обеспечивает обработку HTTP-запросов и предоставляет REST API для взаимодействия с клиентской частью. Данные хранятся в локальной базе SQLite, доступ к которой организован через ORM Sequelize — это упрощает работу с базой данных и позволяет описывать модели данных прямо в коде на JavaScript. Для аутентификации пользователей применяется механизм JWT-токенов, а пароли хешируются с помощью библиотеки bcryptjs, что обеспечивает базовую безопасность системы.

Клиентская часть написана на React с использованием TypeScript, что помогает отлавливать ошибки на этапе разработки и делает код более предсказуемым. Интерфейс построен на базе библиотеки Material UI, которая предоставляет готовые компоненты в стиле Material Design. Для маршрутизации между страницами используется React Router, а сборка проекта выполняется с помощью Vite — современного инструмента, который обеспечивает быструю разработку и оптимизированную сборку для production. Drag-and-drop функциональность для перетаскивания задач между колонками реализована через библиотеку hello-pangea/dnd, которая является продолжением популярного react-beautiful-dnd.

## Функциональные возможности

В системе реализована регистрация и авторизация пользователей, после входа открывается главная страница с дашбордом проектов. Пользователь может создавать новые проекты, каждый из которых имеет название и описание. Внутри каждого проекта организована Kanban-доска с тремя колонками: "К выполнению", "В работе" и "Выполнено". Задачи можно создавать, редактировать и перемещать между колонками простым перетаскиванием мышью, при этом их статус автоматически обновляется.

Каждая задача содержит заголовок, описание и может быть назначена на конкретного пользователя. К задачам можно добавлять комментарии, что позволяет вести обсуждение непосредственно в контексте работы. Одна из интересных функций — встроенный учёт времени: для любой задачи можно запустить таймер, который будет отсчитывать время работы, или добавить запись о потраченном времени вручную. Все записи времени сохраняются и отображаются в интерфейсе задачи, что позволяет отслеживать фактические трудозатраты.

Интерфейс приложения адаптирован под современные требования: использована тёмная тема Material UI, все формы валидируются на клиенте, а при взаимодействии с сервером показываются уведомления об успехе или ошибках. Навигация организована через защищённые маршруты, так что неавторизованные пользователи перенаправляются на страницу входа.

## Как запустить проект локально

Для запуска приложения потребуется установленный Node.js версии 16 или выше. Проект состоит из двух частей — backend и frontend, которые нужно запустить отдельно.

Сначала настройте backend. Перейдите в папку backend и установите зависимости командой npm install. Затем создайте файл .env в этой же папке, скопировав содержимое из env.example. В файле .env укажите секретный ключ для JWT (параметр JWT_SECRET можно задать любой строкой) и убедитесь, что параметр DB_DIALECT установлен в значение sqlite. После этого запустите сервер командой npm run dev — он станет доступен по адресу http://localhost:4000.

Для frontend аналогично перейдите в папку frontend, выполните npm install, создайте файл .env на основе env.example. В этом файле укажите VITE_API_URL=http://localhost:4000/api, чтобы клиент знал, куда отправлять запросы. Запустите frontend командой npm run dev, после чего приложение откроется в браузере по адресу http://localhost:5173.

## Сложности и перспективы развития

В процессе разработки наиболее интересной задачей оказалась реализация drag-and-drop для задач на доске — пришлось разобраться с работой библиотеки перетаскивания и синхронизацией состояния между клиентом и сервером при изменении позиций задач. Также потребовалось продумать логику учёта времени, чтобы правильно считать длительность работы и корректно отображать активный таймер в интерфейсе.

В дальнейшем проект можно расширить добавлением ролевой модели для совместной работы нескольких пользователей в одном проекте, реализацией уведомлений о новых комментариях или изменениях в задачах, а также экспортом отчётов по времени в различных форматах.
